# VRChat 无障碍操作设计指南

> 来源: VRCLibrary (Code-Floof) | 更新: 2026-06-05

## 视觉无障碍

### VR 硬件差异

不同头显有不同的色度和黑阶：
- **LCD (Index)**：画面表现良好
- **OLED (Vive)**：黑阶优秀，但黑暗区域物体难以识别

### 散光

> ⚠️ 散光会导致光源出现晕轮或光斑，类似 Bloom/Lens Flare 效果

**症状**：
- 模糊或扭曲视觉
- 眼疲劳/不适
- 头痛
- 夜视力障碍
- 眯眼

**解决方案**：增大 **Gamma 值**可有效降低眩光（使明暗区域更清晰）

### 晕动症

**加剧因素**：
- 未经玩家操控的视角移动
- 不断变化的场景
- 失去参照物（地平线/地面）
- 画面扭曲变形
- 画面帧数较低

### 癫痫

**触发因素**：
| 类别 | 具体内容 |
|------|----------|
| 视觉特效 | 大量躁动特效、LTCGI、舞台特效、上帝之光 |
| 闪烁 | 闪烁灯光（不论颜色/亮度/模式） |
| 后处理 | 强烈后处理效果 |
| 图案动画 | 特定图案/动画 |
| 纹理 | 滚动纹理（如水） |

**设计建议**：允许玩家自定义或禁用某些效果

### 过度刺激

**症状**：
- 焦虑/紧张
- 感觉需要逃离
- 失去控制感
- 头晕/恶心
- 难以处理信息
- 恐慌发作

**解决方案**：减少视觉信息量

### 色盲

**设计建议**：
- 使用强烈的对比颜色
- 避免相同颜色的渐变（相似颜色物体看起来相同）

### 有限视野

- 大多数 VR 头显 FOV 相同，但镜片/矫正镜片可能限制视野
- HUD 界面元素应在中心 30 度圆周内清晰可读
- **解决方案**：UI 移动时保持固定，静止时可环绕查看

## 音频无障碍

### 听觉障碍

**设计建议**：
- 视觉提示应伴随音频提示
- 单耳耳聋：音频提示应在双耳播放
- 特定频率听力丧失：音频覆盖广泛频率范围

### 容易受惊

> ⚠️ 避免突然的、高强度的音效

即使目的是吓唬玩家，也应提供警告或可跳过选项。

## 物理无障碍

| 类型 | 说明 |
|------|------|
| **空间限制** | 低矮天花板/狭小空间限制活动 |
| **硬件限制** | 追踪准确度受方向/地面影响 |
| **身体限制** | 无法弯腰/伸展（膝盖不便/单手用户） |

**设计建议**：确保行动不便的玩家能完成操作，或明确告知所需要求。
