# 遮挡剔除（Occlusion Culling）深度指南

> 来源: VRCLibrary (Silent) | 更新: 2026-06-05

---

## 核心概念

### 遮挡剔除的作用

- **仅作用于视觉层面** — 减少 Draw Calls，不影响逻辑处理
- **与距离无关** — 无论距离多远，只要被判定为不可见就会被隐藏
- 配合区域禁用可进一步提升性能

### 对象分类

| 类型 | 说明 |
|------|------|
| **静态遮挡物（Static Occluders）** | 不会移动的网格/地形，可遮挡后方对象 |
| **静态被遮挡物（Static Occludees）** | 可被遮挡的静态对象 |
| **动态被遮挡物（Dynamic Occludees）** | 未标记静态的对象，启用 Dynamic Occlusion 后可被隐藏 |

> ⚠️ 动态遮挡精度低于静态遮挡，只有大型实心对象才能有效隐藏动态对象

## 推荐设置

### 遮挡物规则

| 适合作为遮挡物 | 不适合作为遮挡物 |
|----------------|------------------|
| 大型实心对象 | 树木（大量孔洞） |
| 墙壁/地板 | 边缘复杂的对象 |
| — | 透明材质（不产生遮挡） |

### 被遮挡物规则

| 规则 | 说明 |
|------|------|
| 始终可见对象不设为被遮挡物 | 天空盒、巨型海洋等会产生误判 |
| 小型对象设为被遮挡物 | 减少渲染开销 |

### EditorOnly 假遮挡物

> 💡 可使用 EditorOnly 标签创建"假遮挡物"（不渲染但参与烘焙）

用法：
- 创建大型碰撞体
- 标记为 EditorOnly
- 放置在区域边界
- 游戏运行时不可见，但遮挡烘焙时有效

## 遮挡区域设置

### 视景体（View Volumes）

- 默认整个场景作为单一大型视景体
- **建议**：为每个独立区域添加遮挡区域，减少数据量

### 烘焙参数

| 参数 | 说明 | 建议值 |
|------|------|--------|
| **Smallest Hole** | 识别孔洞的阈值 | 0.05-0.5 米 |
| **Smallest Occluder** | 细节保留程度 | 2-6 米（略大于玩家尺寸） |

> 💡 粗略设置用于编辑，精细设置用于最终优化

## 案例：The Devouring 大型地图

### 实践技巧

| 技巧 | 说明 |
|------|------|
| 墙壁内置遮挡盒 | 简化设置和烘焙流程 |
| 大型遮挡方块分隔区域 | 使用 EditorOnly 模式 |
| 装饰物放置在内部 | 避免闪烁问题 |
| 虚拟遮挡物略小于暴露边缘 | 避免可见性问题 |
| 内部看到粉红色 | 存在缺口，需要修复 |

### 世界设计建议

- 不需要开放空间时 → 建造墙壁
- 走廊使用转角/Z字形布局
- 阻挡大型开阔视野
- **避免一次性看到过多内容** — 多人中尤为重要

## 遮挡门（Occlusion Gates）

可开闭的遮挡机制，配合大型可动物体（如门）使用。

- 门关闭时激活遮挡门
- 享受遮挡剔除性能提升

## VR 环境注意事项

> ⚠️ Unity 2018 VR 环境中可能遇到"闪烁"问题，尚未找到解决方案

### 排查方法

| 现象 | 含义 |
|------|------|
| 内部看到粉红色 | 可能有错误的遮挡反应 |
| 外部看不到粉红色 | 遮挡中存在缺口 |
