# Unity 反射探针与 VRChat World


---

## 概述

Unity 反射探针是烘焙光照的重要组成部分，用于提供准确的反射效果。

### 工作原理
1. 反射探针是 360 度拍照的摄像头
2. 拍摄周围环境六个面，拼接成立方体贴图（CubeMap）
3. 6个面自动组合成 512x512 等规格的纹理
4. 系统自动生成模糊版本（用于粗糙材质）

---

## 核心约束

### 双探针混合机制
- 物体使用**最近的 2 个**反射探针
- 根据距离自动混合效果
- 仅一个探针时直接使用该探针
- 无探针时回退到**环境反射**（默认天空盒）

### 静态合批限制
- 只能合并受**同一个探针影响**的物体
- 不同探针影响的物体会分批渲染
- **解决方案**:
  - 统一 Anchor Override
  - 手动合并模型

### 动态物体 Anchor 陷阱
- 默认锚点是**网格原点**（建模软件中的 0,0,0），非模型中心
- 可能导致反射不准确
- **解决方案**: 使用 Anchor Override 指定锚点

### Avatar 锚点建议
- 大部分 Avatar 使用 **Chest** 作为所有子物体的 Anchor
- 保证身体各部位时刻处于同样亮度
- 避免身体亮度分裂问题

---

## 探针类型

### 普通探针
- 从中心向外渐变混合
- 适合开放空间

### 箱形投影探针（Box Projection）
- 从边缘向中心混合
- 适合**封闭空间**（房间、内部场景）
- 根据几何形状自动调整反射
- **注意**: 大物体会导致边界被拉伸，需单独设置锚点或拆分网格

### Custom + 默认反射
- 创建 Custom 类型探针
- 在 Lighting Window 设置为环境反射源
- 可禁用原始探针
- **优点**: 不影响批处理性能

### 实时探针
- 游戏内实时渲染，不保存纹理
- 适合光照/物体频繁变化的场景
- **缺点**: 每个 512x512 约 2MB，存储开销大

---

## 设置流程

### 基础探针设置
1. 创建 Reflection Probe，Type 改为 "Custom"
2. 根据场景内容选择是否启用"动态对象"
3. 烘焙生成反射贴图
4. 打开 Lighting Window 设置环境反射源为"自定义"
5. 拖入探针，可禁用原始探针

### 复杂场景探针放置
1. 识别反射不匹配的区域
2. 在问题区域添加额外探针
3. 调整探针影响范围
4. 封闭空间启用箱形投影

## 性能优化技巧

### 超采样技术
**问题**: 低分辨率烘焙出现锯齿/模糊

**解决方案**:
1. 以更高分辨率烘焙探针
2. 导入时将纹理最大尺寸设置为较低值
3. Unity 自动执行高质量抗锯齿压缩

### 探针区域控制
- 调整探针边界控制影响范围
- 边界处自动插值混合
- 探针边界会"呼吸"——根据周围物体自动调整

## 镜面高光问题

### 问题原因
- 点光源/聚光灯在反射探针中没有物理实体
- 光源附近无高亮物体时，反射中无对应高光

### 解决方案

#### 工具方案: SpecularProbes
- GitHub: https://github.com/zulubo/SpecularProbes
- 自动在每个点光源添加发光球体
- 在反射探针中创建闪亮高光

#### 着色器方案
- 使用支持烘焙镜面高光的着色器
- 如 Bakery 的"光图镜面"模式

---

## 时变光照处理

### 探针切换模式
- 反射探针与组件关联
- 启用/禁用探针控制其生效
- 不同光照条件使用不同探针

### 实时探针模式
- 用于频繁变化的光照/物体
- 无需存储空间存储多组探针
- 性能开销较大

## 最佳实践总结

1. **封闭空间** → 启用箱形投影
2. **Avatar 场景** → 使用 Chest 作为 Anchor Override
3. **批处理优化** → 统一探针影响区域
4. **存储优化** → 考虑实时探针替代多组烘焙探针
5. **视觉质量** → 使用超采样技术
